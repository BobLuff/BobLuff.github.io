使用hexo和github pages搭建一个简易blog系统

一、前置条件

1.1 github账号

首先，在github上创建一个账号，新建一个名为用户名.github.io的仓库。

在设置（setting）中将它发布为github pages。

正如官网介绍，GitHub Pages是一个合适个人开发者的网站（Websites for you and your projects.）。

1.2 node.js+hexo

1）node.js和npm

node.js是一个基于Chrome V8引擎的JavaScript运行环境,它是一个轻量级的Web服务器。npm(Node Package Manager)是node,js内置的一套包管理工具,我们可以使用npm自助下载依赖.

安装完node.js后,使用npm命令安装Hexo

  • -g 表示全局安装
  • hexo-cli hexo Command Line Interface hexo的命令行模式工具

2）Hexo介绍

Hexo是一个静态博客生成工具,它是一个快捷简单的博客框架,具备编译MarkDown文本,转换为HTML文本,上传文件到git等功能.

3）Hexo常用方法:

  • 查看已安装模块版本
  • 创建一篇文章

二、搭建blog项目

2.1 新建本地hexo项目

首先.在某一空文件夹中,调出命令行串口，输入如下命令，初始化项目，搭建hexo站点

  • node_modules：依赖包文件夹
  • public：存放的是生成的页面
  • scaffolds：命令生成文章等的模板
  • source：用命令创建的各种文章，我们编写的markdown文件即存放在该目录
  • themes：主题，blog的样式，可从网上下载自定义样式
  • _config.yml：整个博客的配置
  • db.json：source解析所得到的
  • package.json：项目所需模块项目的配置信息
  • .deploy_git ： 编译后的部署文件

2.2 配置项目

项目根目录中的_config.yml文件主要用来配置博客的各类样式，如菜单栏、标签页等。

1）设置github地址

在deploy中配置需要发布的github地址

2）其他配置

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
keywords 网站的关键词。支持多个关键词。
author 您的名字
language 网站使用的语言。对于简体中文用户来说，使用不同的主题可能需要设置成不同的值，请参考你的主题的文档自行设置，常见的有 zh-Hans和 zh-CN。
timezone 网站时区。Hexo 默认使用您电脑的时区。请参考 时区列表 进行设置，如 America/New_York, Japan, 和 UTC 。一般的，对于中国大陆地区可以使用 Asia/Shanghai。

具体配置请参考hexo官方文档（配置 | Hexo）

主题就是blog的样式。在网上可以搜索到各种类型的主题样式，hexo也提供的可供下载的主题（Themes | Hexo）。
